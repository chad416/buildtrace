# BuildTrace Internationalization

## Phase

Phase 0 — Professional project foundation + security docs.

## Supported locales

BuildTrace Beta supports these locales from day one:

- en — English
- cs — Czech
- sk — Slovak
- pl — Polish
- de — German
- fr — French
- es — Spanish

## Locale routing

The web app uses locale-based routes:

- /en
- /cs
- /sk
- /pl
- /de
- /fr
- /es

The root route redirects to /en.

## Translation rule

No user-facing UI text should be hardcoded in later product screens.

User-facing text should use translation keys.

This includes:

- navigation
- buttons
- form labels
- validation messages
- empty states
- loading states
- error states
- status labels
- document category labels
- QR portal labels
- emails
- PDF export labels

## Uploaded document language

The SaaS UI is multilingual.

Uploaded technical documents are not automatically translated in beta.

Document language must later be stored as metadata:

- en
- cs
- sk
- pl
- de
- fr
- es
- unknown

## Phase 0 implementation

Phase 0 includes:

- shared locale constants
- i18n package
- seven message JSON files
- static locale routes in the web app
- route validation for unsupported locales

## Known Phase 0 gaps

Phase 0 does not yet include:

- full next-intl provider wiring
- language switcher
- translated dashboard navigation
- translated forms
- translated emails
- translated PDFs

These are planned for later phases.
