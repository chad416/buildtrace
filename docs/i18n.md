# BuildTrace Internationalization

## Current phase

Phase 1 - Industrial UI shell + multilingual UI skeleton.

Phase 1 has started, but it is not complete.

## Supported locales

BuildTrace Beta supports these locales from day one:

- en - English
- cs - Czech
- sk - Slovak
- pl - Polish
- de - German
- fr - French
- es - Spanish

## Locale routing

The web app uses locale-based routes:

- /en
- /cs
- /sk
- /pl
- /de
- /fr
- /es

The root route redirects to `/en`.

Unsupported locale routes are validated and should return `notFound()` according to the current route rules.

## Translation rule

No user-facing UI text should be hardcoded in product screens.

User-facing text should use translation keys.

This includes:

- navigation
- buttons
- form labels
- validation messages
- empty states
- loading states
- error states
- status labels
- document category labels
- QR portal labels
- emails
- PDF export labels
- language switcher labels

## Phase 0 implementation

Phase 0 included:

- shared locale constants
- i18n package
- seven message JSON files
- static locale routes in the web app
- route validation for unsupported locales
- root redirect to `/en`

## Phase 1 implementation so far

Phase 1 currently includes the first committed baby step:

- `LanguageSwitcher` component

- translated language labels in all 7 message JSON files
- `appMessages` export from `packages/i18n`
- language switcher mounted on the localized landing route

Latest related feature commit:

- `951e43e feat(web): add multilingual language switcher`

## Language switcher behavior

The current language switcher:

- renders one link per supported locale
- uses labels from translation message files
- uses the shared `locales` constant from `@buildtrace/i18n`
- marks the current locale with `aria-current="page"`
- uses locale routes such as `/en`, `/cs`, `/de`, and `/es`
- does not hardcode visible language labels inside the component

Manual runtime verification passed:

- localized page loaded
- language switcher was visible
- switching from Spanish to English worked
- supported locale routes loaded without crash

## Uploaded document language

The SaaS UI is multilingual.

Uploaded technical documents are not automatically translated in beta.

Document language must later be stored as metadata:

- en
- cs
- sk
- pl
- de
- fr
- es
- unknown
